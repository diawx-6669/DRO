# DRO — Dispatch Route Optimizer

Диспетчерская платформа для оптимизации вывоза вторичного сырья (пластика, ПЭТ-тары).

Проект подготовлен для Республиканского молодежного хакатона «Tech Vision 2026»
(направление **Smart City & GreenTech**, трек **Waste Management**).

---

## 1. Зачем это нужно

**Кому.** Владелец малого или среднего бизнеса по сбору и приёму вторсырья: у него
есть сетки-контейнеры по городу и один-два мусоровоза.

**Проблема.** Владелец не знает, какие баки заполнены, а какие — нет. Из-за этого он
либо гоняет машину впустую к полупустым контейнерам (жжёт бензин и время), либо
узнаёт о переполненном баке только когда пластик уже разлетелся по двору и пришла
жалоба от жителей или штраф от акимата.

**Решение.** DRO подключает к каждому баку датчик, который следит за уровнем
заполнения, и превращает эти данные в готовый маршрут для водителя: система сама
решает, какие баки уже пора объезжать, и строит для них кратчайший путь.

**Эффект.** Меньше пустых рейсов и топлива, ноль сюрпризов с переполненными баками,
чище дворы.

---

## 2. Как это работает — по шагам

```
Датчик на баке  →  Backend (сервер)  →  База данных  →  Карта диспетчера  →  Маршрут водителю
```

1. **Датчик** на крышке контейнера измеряет расстояние до кучи мусора (ультразвуком)
   и передаёт число (% заполнения) на сервер.
2. **Сервер** сохраняет это значение в базу данных и присваивает баку статус:
   - 🟢 зелёный — заполнен до 50%, трогать не нужно
   - 🟡 жёлтый — 51–80%, скоро понадобится
   - 🔴 красный — больше 80%, нужно вывозить в первую очередь
3. **Карта** в диспетчерской (сайт) в реальном времени показывает все баки города
   цветными точками — диспетчер видит картину по всему городу одним взглядом.
4. По кнопке **«Построить маршрут»** сервер берёт все красные баки, считает
   кратчайший путь объезда (алгоритм VRP — Vehicle Routing Problem, тот же класс
   задач, что используют доставки и такси) и выдаёт готовую последовательность
   адресов с итоговым километражем.
5. Этот маршрут передаётся водителю — он просто едет по списку точек.

## 3. Что показывать жюри (демо)

Сайт (диспетчерская): **ссылка на твой деплой Vercel**

- **Верхняя полоса** — счётчики баков по статусам (сколько требуют вывоза прямо сейчас)
- **Карта** — цветные маркеры баков по городу, клик по маркеру показывает адрес и %
- **Список контейнеров справа** — те же баки, отсортированы по убыванию заполнения
- **Кнопка «Построить маршрут»** — жмёшь, и сервер в реальном времени считает
  оптимальный маршрут по всем красным (переполненным) точкам, рисует его линией
  на карте и выводит итоговый километраж

**Демонстрация живого датчика:**
Прямо в диспетчерской есть кнопка **«Проверка датчика →»** в правом верхнем углу шапки —
открывает окно поверх карты для проверки датчика вживую перед жюри, без перехода
на отдельную страницу:

1. Друг с настоящей Arduino подключает к ней датчик расстояния (HC-SR04)
2. Открывает твою ссылку на сайт на своём ноутбуке в Chrome или Edge
   (Safari и Firefox нужную технологию не поддерживают)
3. Подключает Arduino к ноутбуку по обычному USB-кабелю
4. Жмёт «Проверка датчика →» в шапке сайта, в открывшемся окне — «Подключить Arduino»,
   выбирает нужный порт
5. Прямо в этом окне в реальном времени появляются живые показания расстояния с датчика

Это доказывает жюри, что железная часть проекта реальна и работает, не только
макет на экране — и всё происходит в одном интерфейсе, без открытия других вкладок.

## 4. Архитектура и стек технологий

```
├── backend/       Python (FastAPI) — API, база данных, алгоритм маршрутизации
├── frontend/       React — диспетчерская панель (сайт, который видит жюри)
├── hardware_iot/   Прошивки для датчиков (ESP32 по Wi-Fi, Arduino Uno по USB)
└── docs/           Документация
```

| Слой | Технология | Зачем |
|---|---|---|
| Frontend | React + Vite, Leaflet.js | Интерактивная карта и диспетчерский интерфейс |
| Backend | Python, FastAPI | API, приём данных с датчиков, бизнес-логика |
| Алгоритм маршрутов | Google OR-Tools | Решает задачу VRP — кратчайший объезд контейнеров |
| База данных | PostgreSQL | Хранение контейнеров и истории показаний |
| IoT | ESP32 + HC-SR04 (по Wi-Fi) или Arduino Uno (по USB) | Железная часть — датчики заполнения |
| Деплой | Vercel (frontend) + Railway (backend + БД) | Хостинг для демо |

## 5. Быстрый старт (для разработки)

### Backend
```bash
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload
```

### Frontend
```bash
cd frontend
npm install
npm run dev
```

### Тестовые данные
Чтобы сразу увидеть баки на карте без реального датчика:
```bash
curl -X POST https://<твой-backend>.up.railway.app/api/bins/seed
```
Добавит 8 тестовых контейнеров вокруг Астаны с разным уровнем заполнения.

## 6. Экономический и социальный эффект

- **Для бизнеса** — меньше расходов на топливо и амортизацию машин, больше собранного
  сырья за счёт того, что не тратится время на пустые рейсы.
- **Для города** — чистые дворы без разлетающегося пластика, меньше жалоб жителей и
  штрафов, снижение углеродного следа от лишних поездок мусоровозов.
